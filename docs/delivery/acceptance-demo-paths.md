# 验收演示路径清单

生成时间：2026-06-01

## 演示前准备

1. 确认 Docker Desktop 已启动。
2. 在 `03_dev` 目录启动 MySQL：

```powershell
docker compose up -d mysql
```

3. 在 `03_dev` 目录启动四端服务：

```powershell
npm run dev
```

4. 打开：

| 端 | 地址 |
|---|---|
| 学生端 | `http://localhost:5173` |
| 咨询师端 | `http://localhost:5174` |
| 管理员后台 | `http://localhost:5175` |
| 后端健康检查 | `http://localhost:3000/health` |

## 核心联动演示建议

| 闭环 | 演示方式 |
|---|---|
| 预约闭环 | 学生端提交预约，咨询师端处理，管理员端查看预约状态。 |
| 风险闭环 | 学生端提交风险筛查，咨询师端风险记录和管理员端风险预警同步可见。 |
| 调班闭环 | 咨询师端提交调班，管理员端审批，咨询师端消息/排班同步。 |
| 转介闭环 | 咨询师端发起转介，接收端处理，管理员端可查看。 |
| 内容闭环 | 管理员端发布文章/活动，学生端资讯/活动列表可见。 |

## 通过标准

- 学生端和咨询师端保持移动端小程序风格。
- 管理员端保持 PC 后台风格。
- 页面不空白，不出现阻塞性错误。
- 核心入口没有缺失。
- 浏览器控制台无 `error` 级阻塞日志。
