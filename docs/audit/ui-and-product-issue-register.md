# UI 与产品问题清单

## 2026-06-15 更新

本轮已修复/降级：

- P1：学生端、咨询师端、管理员 H5 不再使用 `localStorage` 持久化 access token；改为内存 access token + HttpOnly refresh cookie。
- P1：管理员复杂表格在 768px 以下切换为卡片式行布局，390px 移动 dashboard 和学生管理页未检测到横向溢出。
- P1：学生端 H5 首次登录强制改密浏览器验证通过，完成后进入业务首页。
- P1：咨询师端 H5 首次登录强制改密浏览器验证通过，完成后进入工作台。
- P1：小程序 H5 生产构建首屏在 390x844 下无横向溢出。

仍需跟进：

- P1：浏览器 E2E 仍缺少完整预约创建、取消、咨询师确认/签到/完成、管理员查看状态的自动化路径；目前由 API 测试覆盖。
- P1：原生微信小程序真机安全区、键盘顶起、授权和存储行为未验证。
- P1：管理员敏感批量操作、删除/禁用/结案等动作仍缺少统一二次确认策略。
- P2：学生端、咨询师端和小程序 H5 的品牌视觉仍未完全统一。

审查日期：2026-06-14

分级：

- P0：阻断启动、核心业务闭环或演示。
- P1：成品交付前应修复。
- P2：质量、体验或可维护性改进。

## 跨项目

### P0

- 当前未发现启动、构建或核心 API 闭环阻断项。

### P1

- 学生、咨询师和管理员入口文件仍然过大，页面级回归风险高。
- H5 使用浏览器 `localStorage` 保存 JWT。正式上线前应迁移到更稳健的会话方案，并配合 CSP/XSS 防护。
- 尚未实现服务端会话撤销、短期令牌轮换和管理员强制下线机制。

### P2

- Windows PowerShell 的部分工具输出仍可能发生 UTF-8 显示乱码。
- 多端缺少自动化端到端浏览器测试，当前截图验证仍由人工命令驱动。

## 学生端 H5

### P0

- 无。

### P1

- 固定手机壳布局适合原型展示，不是完整响应式 H5；桌面端会保留模拟设备边框。
- 首页课程和部分推荐文案仍含静态展示内容，需明确数据来源或改为空状态。
- 预约咨询师卡片在重复 API 测试后会出现同名测试咨询师，演示库需要隔离或重置。

### P2

- 登录页与小程序品牌视觉仍不一致。
- 紧急帮助入口已显示，但未登录时仍只是说明提示，尚未直接进入完整应急信息页。

### 本轮已修复

- P1：协议勾选从静态 `checked` 改为真实状态，并随登录请求提交。
- P1：移除不可用的账号激活原型和主页“切换咨询师端原型入口”悬浮按钮。
- P1：登录期间显示禁用和“登录中”状态。
- P2：扩大协议勾选控件并提高文字可读性。
- P1：首次登录强制进入改密页，完成前不会加载业务页面。
- P1：弱密码、两次输入不一致和服务端拒绝均有明确反馈。

## 咨询师端 H5

### P0

- 无。

### P1

- 品牌视觉仍与小程序、管理员后台不完全一致。
- 部分页面仍使用 `prototypeAppointments` 等迁移期命名，后续拆分时应统一为业务模型命名。
- 风险记录、转介和文章流程仍需逐页完成移动端截图验收。

### P2

- 登录页仍为固定手机容器，不是完整响应式布局。

### 本轮已修复

- P1：移除未接入真实认证的“微信扫码登录”按钮，改为管理员支持说明。
- P1：补充登录加载和禁用状态。
- P1：首次登录强制进入改密页，完成后才加载咨询师工作台。

## 管理员后台

### P0

- 无。

### P1

- 敏感操作仍缺少二次确认策略，例如批量禁用、删除基础数据、关闭风险记录。
- 管理员移动端已消除横向溢出，但复杂表格在手机上仍以横向信息密度为主，后续应补充卡片式移动列表。

### P2

- 登录页仍偏品牌插画风格，与高密度业务后台存在视觉差异。
- 导航模块较多，后续应按账号权限裁剪可见菜单。

### 本轮已修复

- P1：默认不再勾选“记住我”。
- P1：移除没有真实恢复流程的“忘记密码”链接，改为联系管理员说明。
- P1：登录按钮增加加载和禁用状态。
- P1：新增、导入和重置账号改为一次性随机临时密码，并提供安全交付提示。
- P1：dashboard 趋势、咨询类型、预约状态和最近预约改为后端真实聚合数据，并补充空状态、加载态、错误重试。
- P1：管理员后台 390px 视口由 `scrollWidth=1024` 修复为 `scrollWidth=390`，消除移动横向溢出。
- P1：管理员重置学生/咨询师密码后旧 token 立即失效。

## 微信小程序

### P0

- 无。

### P1

- 仍需在微信开发者工具和真机验证状态栏、安全区、键盘顶起和授权行为。
- uni-app alpha 构建链存在 9 个高危依赖审计项，直接修复需要整套升级和回归。
- 首页右上角图标按钮在 H5 预览中较紧凑，需在 320px 真机视口继续确认边界。

### P2

- 登录页品牌与学生/咨询师 H5 尚未统一。
- 空状态较完整，但部分图标仍来自位图资源，后续应统一图形语言。

### 本轮已修复

- P1：缩短登录 Hero 高度，缓解小屏首屏表单受压。
- P1：协议行、勾选框、紧急帮助和角色切换入口扩大到接近 44px/88rpx 的触控尺寸。
- P1：真实学生账号可通过 H5 预览登录并进入首页。
- P1：新增全局强制改密页和路由守卫，刷新或直接访问业务页也会回到改密流程。
- P2：H5 预览增加品牌 favicon，消除无关 404。

## 2026-06-15 E2E 后 UI/产品问题更新

本轮已修复：

- P1 学生端：创建预约后取消预约，列表和成功卡片会重新同步服务端状态，不再展示已取消预约为待确认。
- P1 学生端：主页顶部工具按钮、预约卡片操作按钮等主要触控区补齐到不低于 44px。
- P1 咨询师端：预约详情和处理按钮改为真实预约数据，不再只操作静态原型内容。
- P1 咨询师端：确认、签到、完成预约按钮补齐触控面积，并通过 E2E 覆盖完整处理路径。
- P1 管理员端：移动端 dashboard 和复杂表格容器修复横向溢出，`390x844` 与 `320x700` 视口通过 E2E 横向溢出检查。
- P1 管理员端：顶部通知、用户区域等主要点击区域补齐到 44px。
- P1 微信小程序 H5：登录、首次改密、首页渲染加入 E2E，检查 320/390/1365 视口横向溢出。

仍需跟进：

- P1 微信小程序：原生 mp-weixin 真机尚未验证安全区、键盘顶起、微信授权、网络失败和持久化登录态。
- P1 管理员端：批量禁用、删除、结案、重置密码等敏感操作仍需统一二次确认和撤销/审计说明。
- P1 全端：网络失败、重复提交、加载态、空状态和错误态尚未全部纳入浏览器自动断言。
- P2 全端：学生端、咨询师端、小程序 H5 的品牌色、图标语言、页面密度仍不完全统一。
- P2 E2E：当前主要检查横向溢出和触控面积，后续应加入视觉回归截图基线和可访问性扫描。

当前页面级风险：

- 学生端：固定手机壳布局仍更偏演示原型，不是完整响应式 Web 产品。
- 咨询师端：部分页面命名仍保留 prototype 迁移痕迹，后续拆分组件时应统一业务命名。
- 管理员端：移动端可读性已经改善，但复杂管理表格仍需继续转为卡片式详情/折叠布局。
- 微信小程序：H5 预览通过不等于真机通过，必须单独验收。
